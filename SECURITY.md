# Security Policy

## APIキーの扱い

- OpenAI APIキーは `services/local-proxy/.env` でローカル管理してください。
- APIキーをソースコード、コミット、Issue、ログに出さないでください。
- `OPENAI_API_KEY` を含む `.env` は Git に含めません。

## ログ運用

- 本リポジトリのローカルプロキシはキーをログマスキングします。
- ただし、プロンプトや面談内容を長期保存する設計ではありません。
- デバッグ時も機密情報の出力を避けてください。

## データ保存

- 音声・文字起こし・質問・回答案は外部サーバーに保存しません。
- プロフィールは `Application Support` に AES-GCM で暗号化保存し、鍵は Keychain 管理です。

## 事故時の対応

1. APIキー漏えい疑いがある場合は即時ローテーション
2. 不要なログファイルを削除
3. 公開済み履歴に機密がある場合は履歴修正と再発防止策を実施
